# 更新日志

## 0.5.1 - 2026-07-22

- 修复嘉立创 EDA 专业版 3.2.166 中 `getDocumentsPath()` 因客户端配置响应结构不一致返回空字符串、导致首次自动备份被误报为未授予外部交互权限的问题；在官方权限检查成功后，从同一客户端配置调用链读取实际文档目录。
- 兼容 `createDirectoryInFileSystem()` 的 `folderPath/path` 参数错配，官方接口返回 `false` 时通过受权限门控的客户端调用链重试；继续拒绝空路径、相对路径、控制字符和畸形文件 URI，并把合法 `file:` URI 转换为文件 API 所需的本机路径。
- 区分宿主 API/权限不可用与宿主目录格式异常，正式日志也记录脱敏后的路径类型、长度和失败阶段；补充默认目录回退、URI 转换、目录拒绝和错误分类回归测试。

## 0.5.0 - 2026-07-22

- 新增当前 SCH/PCB 与外部 BOM 库存校验、板数倍率、缺料筛选和 CSV 导出；支持通用 CSV/TXT/XLS/XLSX 表头识别、工作表与列映射、BOM 版本差异及差异导出，文本 BOM 可识别 UTF-8、UTF-16 和 GB18030。BOM 单文件上限为 10 MiB，库存校验/出库只接受 1 个文件，版本差异只接受 2 个；工作簿限制为 32 个工作表、每表 128 列和 10,000 行数据。
- 新增逐条最低库存、补货视图与导出、收藏、列显隐、数据手册链接和结构化仓位；包装码入库保留原文与解析预览，选中图纸器件可精确定位并高亮库存记录，模型失效时可搜索候选并人工确认。
- 新增“BOM 出库批次与撤销流水”，包含显式整批出库、语义重复识别、新生产批次确认、原子扣减和整批撤销；活动批次引用的条目禁止删除或合并，流水明确只覆盖 BOM 出库与撤销，不冒充完整库存审计。
- 新增项目需求快照、重同步差异确认、板数需求聚合、采购建议、采购记录、成本汇总与 CSV 导出；采购日期按 `YYYY-MM-DD` 保存并兼容读取旧时间戳。替代料关系仅由用户在现有库存间确认，并只用于人工候选排序，不参与自动匹配或扣减。当前同一物理板的 SCH/PCB 两份快照仍会重复累计，需用户避免同时保留。
- 库存文档升级到 schema-v9，加入项目、采购、BOM 流水与替代关系数据；桌面端首次设置自动备份时无需输入路径，插件直接在系统“文档”目录下自动建目录并生成固定“最新快照”JSON，试写成功后启用。恢复前执行严格校验与恢复点保护；自动备份或通知失败不会把已经提交的主库存操作误报为失败。
- 新增跨模块容量预算、文档 revision CAS、操作型 IFrame 原生关闭竞态保护及覆盖关键业务规则的自动化回归；订单导入一次最多选择 100 个文件、单文件上限 10 MiB。嘉立创 EDA BETA API、Web/桌面文件系统差异和跨运行实例并发仍需实机验证。

## 0.4.16 - 2026-07-22

- 将订单防重复与库存更新方式拆成两层文案：明确提示“已导入过的订单不可重复添加”，并用“采购入库”“重置数量”“只新增物料”三个带数量结果的选项说明本次导入如何更新已有库存。

## 0.4.15 - 2026-07-22

- 重写订单导入的库存匹配设置文案，明确该选项只决定“订单物料匹配到现有库存时”如何改变数量，并将三个选项改为累加订单数量、用订单数量覆盖库存和保持库存不变，避免被误解为相同订单或相同文件的处理方式。

## 0.4.14 - 2026-07-22

- 修复删除库存后原订单仍被历史文件指纹拦截的问题：订单去重历史现在关联实际库存记录；关联记录被删除或改成其他物料后，对应文件可以重新导入。
- 部分删除一个订单的库存时，只恢复缺失物料，仍存在的物料计为跳过，不会再次累加数量；多个订单曾合并到同一库存记录时，各订单仍可分别恢复自己的数量。
- 库存文档升级为 schema-v5，仅在批次摘要中保存规范化物料身份键和库存 ID 关联，不保存 Excel 行、订单抬头或收件信息；旧 schema-v4 历史在库存缺失时按当前物料状态兼容恢复。

## 0.4.13 - 2026-07-22

- 库存总览新增“删除所选”：可先全选当前筛选结果，再经不可撤销二次确认批量删除；服务端一次校验全部条目 revision 并原子保存，任一记录过期时整批不删除。
- 自动隐藏重新要求 IFrame 失焦与 EDA 主窗口聚焦配对；切换到 VS Code 等外部应用时保留库存总览，只有焦点确实返回 EDA 主窗口才隐藏。
- 修复复制到个人库被官方接口拒绝后，总览操作仍误报成功的问题；失败现在返回准确状态和原因，并以 `warn` 级别写入诊断日志。

## 0.4.12 - 2026-07-22

- 修复库存总览正常 ready 后马上自动消失的问题：移除启动期同步 autofocus，在页面显示 300 ms、EDA 宿主完成焦点收敛后再聚焦搜索框并启用自动隐藏，启动期 blur 不再被当作用户点击主窗口。
- 保留首次稳定聚焦后的直接点击 EDA 自动隐藏，以及原生最小化窗口控制、活动操作保护和旧会话替代逻辑。

## 0.4.11 - 2026-07-22

- 已 ready 的总览如果遇到 EDA 扩展存储上下文被卸载，按宿主会话终止静默收尾，不再显示与用户操作无关的通用错误框。

## 0.4.10 - 2026-07-22

- 修复首次打开库存总览后不操作内部控件、直接点击 EDA 主窗口时无法自动隐藏的问题：总览 ready 前显式建立 IFrame 与搜索框焦点，确保第一次外部点击也会产生可靠的失焦事件。
- 保持 `0.4.9` 已验证的分类管理、详情、确认弹层隐藏规则和实际操作保护；相关性排序规则本轮不再调整，留待单独确认。

## 0.4.9 - 2026-07-22

- 修复库存总览偶发 `Inventory overview panel unavailable: render-failed`：当插件控制器已经重建但 EDA 仍保留旧的隐藏 IFrame 时，先隐藏并关闭失去控制器的旧会话，再创建新会话，避免新请求在旧页面中等待 ready 超时；旧会话只能按 `requestId` 清理自己的 bridge 数据和窗口，不会在稍后超时时破坏新会话。同一控制器内仍直接恢复原窗口和全部浏览状态。
- 自动隐藏改为直接监听总览 IFrame 失焦并检查页面可见性，不再依赖不稳定的“失焦/主窗口聚焦”事件配对；分类管理、详情和确认弹层可以随总览隐藏，只有正在执行的导入或写入操作会阻止隐藏。
- 原生最小化改用 `constricted` 缩窄标题窗；用户点击最小化、恢复或最大化时，宿主窗口控制通知会阻止该次失焦触发自动隐藏，确保小标题窗保留并可直接恢复。
- 修正空搜索“相关性”的语义：在“全部”中按库存状态和全局名称排序，不再把刚查看的二级分类整组排在列表开头；分类分组只由显式“分类顺序”排序控制。
- 为库存总览补充静默诊断跟踪，记录遗留会话处理、请求写入、IFrame 打开、渲染阶段、恢复显示、显式渲染错误和 ready 超时；不会因为正常打开总览而自动弹出日志面板。

## 0.4.8 - 2026-07-22

- 库存总览允许点击 EDA 主窗口后自动隐藏；只调用固定库存总览 IFrame 的隐藏接口，不修改菜单栏或宿主窗口。再次使用原“库存总览”入口会恢复同一会话。
- 自动隐藏只接受紧邻 IFrame 失焦的 EDA 主窗口聚焦事件；拖拽、分类管理、详情/确认窗口和进行中的操作会阻止隐藏，避免文件选择器或子流程误触发。
- 修复从二级分类返回“全部”时可能沿用旧排序状态的问题：分类、当前排序方式和第一页现在原子切换，并对完整结果集重新排序。

## 0.4.7 - 2026-07-22

- 分类导入忽略 EDA 返回的虚拟 `ALL` 根分类；当 `ALL` 包含实际分类或出现在器件分类路径开头时，提升并保留后续真实的一、二级分类。
- 统一分类导入语义与文案：同级同名分类按现有行为合并，只新增缺少的分类，不再描述为“跳过”。
- 明确复制到官方 EDA 个人库依赖仍被官方标为 BETA、且不建议正式使用的 `LIB_Device.copy()`；插件继续以目标库回读验证为成功条件，无法核验时只报告失败。
- 简化诊断日志保留个人库复制的失败原因和非敏感阶段状态，后续可区分官方接口拒绝、API 异常和复制后回读验证失败。

## 0.4.6 - 2026-07-22

- 修复个人库导入的 `already-present` 假成功：C 编号查询结果必须属于嘉立创 EDA 当前工作区个人库并可按 UUID 回读；新复制还必须核对目标库、复制 UUID 和 C 编号或器件身份，任何不一致都不再报告成功。
- 库存详情改为完整字段视图，显式显示并支持复制立创 C 编号；历史记录可从合法的供应商 C 编号回退。库存封装和 EDA 模型封装分别显示，两者都不存在时显示“—”。
- 修复勾选或取消多个元器件后表格跳回顶部的问题，保持当前横向和纵向滚动位置；筛选、切换分类和翻页仍按新结果正常重置。
- 将“导入 EDA 分类”的唯一可见入口放入库存总览的“管理分类”子窗口；导入后保持窗口打开、刷新分类并显示新增、跳过或降级结果，不修改 EDA 顶栏菜单。
- 库存总览启用嘉立创 EDA 原生最小化按钮，临时操作 EDA 主窗口时不销毁当前总览会话；继续保持不自动弹出插件窗口。

- 将 `headerMenus` 逐项恢复为 0.4.3 的原菜单清单；“导入 EDA 分类”不再挂到 EDA 顶栏，入口位置需确认后再提供。
- 新增清单回归测试，逐项锁定既有三个上下文和原分组菜单，避免未确认的入口再次影响宿主菜单。

## 0.4.4 - 2026-07-21

- 修复库存总览拖拽入口过小的问题：表格行非交互区域现在可直接拖动，拖动已勾选行会批量移动全部勾选项，拖动期间展开所有二级目标；分类侧栏明确区分系统分类与用户分类。
- 在总览表格和详情中突出显示库存封装，并在库存封装为空时显示 EDA 模型封装，详情同时保留两者以便核对和分类。
- 新增“导入 EDA 分类”，可幂等导入个人库或收藏库的两级分类且不自动移动库存条目；完整分类树不可用时降级汇总器件已经使用的分类并明确提示限制。
- 个人库复制不再回退写入收藏库；复制返回 UUID 后必须从个人库回读验证，避免“显示已导入但库内找不到”的误报。
- 在 EDA 支持的各编辑上下文增加顶部“库存总览”直达入口；用户仍需在扩展管理器中启用扩展并勾选“显示在顶部菜单”。

## 0.4.3 - 2026-07-21

- 二级分类展开后使用克制的差异底色；器件分类拖动改为 Pointer Events 与 document 级回退监听，不再依赖 EDA 内嵌浏览器不稳定的 HTML5 拖放或指针捕获。
- 批量移动分类改为横向独立的一级、二级选择框，并新增“全选当前筛选结果”和“清除选择”；可先筛选未分类、封装、制造商或关键词，再一次完成批量归类，不自动猜测分类。
- 订单导入改为单个多字段窗口：一次设置默认库存状态和重复策略，在同一窗口完成逐文件预览，并显示解析、EDA 模型匹配和库存写入进度。
- 订单模型匹配按规范化 C 编号去重并限制为 4 个并发查询，进度写入诊断日志时节流；预览失败可重试，同一预览不能并发或重复提交。
- 修正库存备份和诊断日志导出的提示时序：遵循官方 `saveFile(): Promise<void>` 契约，由系统保存对话框处理目的地，不再在用户选择路径前弹出无法验证的保存结果。

## 0.4.2 - 2026-07-21

- 分类管理支持双击分类块原位重命名、同级拖拽排序和横向自动换行；库存总览的一级分类默认预览三个二级分类，并可展开其余分类。
- 元器件名称旁新增拖动手柄和包含名称、编号、数量、当前分类及存放位置的拖动卡片；拖到左侧未分类、一级分类或二级分类即可直接修改分类。
- 打开对应 C 编号的立创商城页面后，在添加表单中记录可取消的商城核对状态；保存时通过宿主会话校验并写入 `user-confirmed` 商城引用，无 EDA 模型的确认商品记为商城来源。

## 0.4.1 - 2026-07-21

- 修复器件已经位于个人库或收藏库时被误报为复制失败的问题；现在按成功处理，不再重复复制。
- 改为优先复制到官方明确提供的当前工作区个人库；个人库不可写或拒绝复制时，再尽力尝试收藏库，并确保任一库 UUID 查询异常都不会阻断另一个目标。
- 按立创 C 编号检查目标库中是否已有器件，避免重复导入；复制失败时区分无目标库、官方接口拒绝和 API 异常，并将不含器件隐私的逐步结果写入诊断日志。

## 0.4.0 - 2026-07-21

- 将库存总览升级为常驻长会话窗口；详情、完整编辑、删除确认、重复项比对和 EDA 模型重匹配均在总览内部的上层模态完成，操作后局部刷新快照并保留搜索、筛选、分页、滚动和选择状态。
- 分类管理改为同窗口双栏结构，左侧维护一级分类、右侧维护当前一级分类下的二级分类，支持新增、重命名、同级排序和删除确认。
- “按立创 C 编号添加”与“添加自定义元器件”统一为一次性多字段表单，集中录入身份、数量类型、分类、存放位置和备注；不再串联多个原生单字段弹窗。
- EDA 查询结果、重复检测和最终合并均使用宿主签发的一次性令牌及 revision 校验；C 编号变化会废弃旧模型结果，首次发现重复记录时不写库存。
- 商城按钮只在当前 C 编号的 EDA 查询未命中或失败后启用；打开商城仅用于人工核对，不会冒充已经自动读取或确认商城数据。
- 订单导入支持一次选择多个 Excel/CSV/JSON 文件；写入前按文件预览订单号、明细数量、重复原因和预计新增/合并/跳过结果。
- 库存文档升级为 schema-v4，原子记录订单导入批次，并使用订单号与 SHA-256 文件指纹阻止历史重复和同批次重复；v1/v2/v3 文档无损迁移。

## 0.3.5 - 2026-07-21

- 订单导入支持立创商城直接导出的 `.xls`/`.xlsx`：自动查找订单详情表头、识别带单位的订购数量、跳过不发货条目，并避免隐藏空白模板工作表造成重复导入；CSV/JSON 继续兼容。
