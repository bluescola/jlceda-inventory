# 嘉立创 EDA 个人元器件库存

面向嘉立创 EDA 专业版 V3 的个人库存扩展。它把立创商城商品信息与嘉立创 EDA 器件模型作为两种独立状态管理，并在原理图设计时优先使用已有库存。

当前版本为 `0.4.3`。扩展不需要开发者自建服务器；源码、CI 和发行文件放在 GitHub，库存写入嘉立创 EDA 的扩展用户配置。

## 已实现

- 输入 C 编号后优先查询嘉立创 EDA 系统器件库；命中时直接使用模型信息，未命中或查询失败时再打开立创商城确认。
- 商城商品与 EDA 模型保持独立；商城有商品但没有 EDA 模型时仍可保存和管理库存。
- “按立创 C 编号添加”和“添加自定义元器件”共用同一个 IFrame 多字段表单，一次填写身份、数量、分类、存放位置和备注，不再串联原生单字段弹窗。
- 在库存列表和详情中分别显示商城确认状态与 EDA 模型状态，并支持稍后重新匹配模型。
- 添加自定义元器件，数量支持真实数量、大概数量和未知数量。
- 库存总览作为常驻父窗口；双栏分类管理、只读详情、完整编辑、删除确认、重复比对和 EDA 模型重匹配均在同一窗口的上层模态中完成，不丢失搜索、筛选、分页、滚动和选择状态。
- 数量必填且必须为非负整数；负数给出明确检查提示，0 自动标记已用完，也可在同一表单中切换已用完状态。
- 修改 C 编号后重新查询 EDA 模型，并将新模型属性作为可逐字段采用的建议；不强制覆盖用户维护的库存信息。
- 编辑产生重复身份时并列展示现有记录和待保存记录，只有用户确认后才合并；保存通过记录 revision 防止覆盖其他电脑上的较新修改。
- 删除前重新读取最新记录，并显示名称、编号和库存数量进行不可撤销确认。
- 将有官方库引用的库存元器件导入个人库，并对已存在于个人库或收藏库的器件进行幂等识别；收藏库仅按官方 BETA 接口能力尽力尝试。
- 使用分类化库存总览管理大量元器件：顶部主搜索框覆盖 C 编号、供应商编号、厂家型号、名称、制造商和封装，左侧支持两级用户分类；器件可直接拖到分类，批量移动使用独立的一级/二级字段，并可全选当前筛选结果一次归类。
- 搜索精确匹配优先于前缀和包含匹配，同等匹配时有库存优先；搜索属于总览本身，不作为菜单项或虚拟元器件显示。
- 在原理图中选择在库元器件并绑定到鼠标放置。
- 批量导入立创商城订单详情 `.xls`/`.xlsx`，在一个多字段窗口中一次设置默认状态和重复策略，预览逐文件订单号、明细、数量和预计库存变化，并显示解析、模型匹配和写入进度；订单号与 SHA-256 文件指纹用于阻止重复导入，同时保留 CSV/JSON 兼容。
- 导出版本化 JSON 备份；保存目的地与覆盖确认由 EDA 系统保存对话框处理。
- 简体中文和英文界面及菜单。

## 同步与服务器

本扩展没有自建后端，也不要求用户配置数据库。库存使用 `SYS_Storage.setExtensionUserConfig()` 保存。

官方文档将其描述为“扩展用户配置”，但没有承诺跨电脑同步、容量、传播延迟或并发冲突规则。因此仓库当前把跨电脑同步标为 **待 V3 双设备验证**，不把它宣传为已经得到官方保证的云数据库。验证步骤见 [跨设备验证清单](docs/cross-device-validation.md)。在验证完成前，请定期使用“导出库存备份”。

GitHub 只能托管源码、CI 构建和 Release 文件，不能直接充当运行时数据库。

## 安装

1. 从 Release 或 CI 构建产物中取得 `jlceda-inventory_v0.4.3.eext`。
2. 打开嘉立创 EDA 专业版 V3。
3. 进入“高级 -> 扩展管理器 -> 导入”，选择 `.eext` 文件。
4. 从顶部“元器件库存”菜单开始使用。

## 当前边界

- 嘉立创目前没有公开的商城订单读取 API，因此订单通过立创商城导出的 Excel 订单详情（或自备 CSV/JSON）导入。
- LCSC 官方商品 API 需要申请 key/secret、请求签名和 IP 授权，商城网页也不允许扩展跨域读取。因此无服务器版本只在 EDA 模型未命中或查询失败时打开官方商城供用户确认，不抓取网页，也不冒充已经自动读取商城数据。
- `LIB_Device`、`LIB_Device.copy()`、文件选择和鼠标放置属于官方 BETA API，已集中隔离在平台层，仍需在浏览器版和客户端实测。
- 统一新增表单和库存总览使用官方 IFrame、扩展用户配置和定时器接口；请求、操作和响应均带会话标识，完成后立即清理临时数据。IFrame API 仍处于 BETA，启动失败会明确报错，不会静默切换成行为不同的逐项输入流程，发布前必须在 Web 和桌面客户端实测。
- 鼠标放置成功仅表示元器件已绑定到鼠标，不表示用户完成放置，因此不会自动扣减库存。
