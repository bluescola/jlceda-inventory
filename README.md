# 嘉立创 EDA 个人元器件库存

面向嘉立创 EDA 专业版 V3 的个人库存扩展。它把立创商城商品信息与嘉立创 EDA 器件模型作为两种独立状态管理，并在原理图设计时优先使用已有库存。

当前版本为 `0.4.12`。扩展不需要开发者自建服务器；源码、CI 和发行文件放在 GitHub，库存写入嘉立创 EDA 的扩展用户配置。

## 已实现

- 对当前原理图、PCB 或外部 BOM 执行只读库存校验，按板数聚合需求，区分充足、不足、库存缺失、需复核、需盘点和身份未匹配，并导出缺料 CSV。
- 读取 CSV、TXT、XLS 和 XLSX BOM，检测文本编码、工作表与表头，支持人工列映射、两份 BOM 的新增/删除/数量/身份差异筛选和 CSV 导出。库存校验或出库只接受 1 个 BOM，版本差异只接受 2 个；单文件上限为 10 MiB，工作簿上限为 32 个工作表、每表 128 列和 10,000 行数据。
- 为单条库存设置最低库存与收藏状态，在总览筛选补货/盘点项、导出补货 CSV、配置显示列，并维护数据手册 URL 与柜/盒/行/列结构化仓位。
- 从当前 SCH/PCB 选中器件精确定位并高亮库存；模型缺失时提供确定性候选搜索和人工确认；包装二维码先展示原文与 `pc`、`pm`、`qty` 解析结果，再进入既有入库草稿与重复合并流程。
- 新增“BOM 出库批次与撤销流水”：通过明确确认执行 BOM 整批原子出库，按语义需求阻止误重复扣减，并允许把相同需求确认为新的生产批次；支持整批撤销和关联流水。该流水只覆盖 BOM 出库/撤销，普通编辑和订单入库不伪装成完整审计流水。
- 保存项目/文档需求快照，重同步前展示差异，按板数生成采购建议；独立维护采购平台、订单、单价、币种和日期并汇总成本。采购日期按 `YYYY-MM-DD` 纯日历日期保存，读取旧时间戳时自动归一化。用户可确认库存条目间的替代关系，候选只用于人工排序，不参与自动扣减。
- 桌面端首次设置自动备份时无需输入或选择路径；插件会直接在系统“文档”目录下创建 `JLCEDA-Inventory/jlceda-inventory-latest.json`，试写成功后启用，后续库存写入更新同一份“最新快照”。已有旧路径可继续使用，也可一键改回默认位置。恢复会严格校验 schema、容量、引用与流水闭合，先保存恢复点，主恢复失败时还原原恢复点。Web 端继续使用手动导出和文件恢复。

- 输入 C 编号后优先查询嘉立创 EDA 系统器件库；命中时直接使用模型信息，未命中或查询失败时再打开立创商城确认。
- 商城商品与 EDA 模型保持独立；商城有商品但没有 EDA 模型时仍可保存和管理库存。
- “按立创 C 编号添加”和“添加自定义元器件”共用同一个 IFrame 多字段表单，一次填写身份、数量、分类、存放位置和备注，不再串联原生单字段弹窗。
- 在库存列表和详情中直观显示库存封装与 EDA 模型封装，分别显示商城确认状态与 EDA 模型状态，并支持稍后重新匹配模型。
- 添加自定义元器件，数量支持真实数量、大概数量和未知数量。
- 库存总览作为常驻父窗口并支持宿主原生缩窄最小化，手动最小化后保留可点击恢复的小标题窗；点击 EDA 主窗口时连同分类管理、详情或确认弹层一起隐藏，只有正在导入或写入的操作会阻止隐藏。再次从原菜单打开会保留搜索、分类、筛选、排序、滚动和选择状态。
- 数量必填且必须为非负整数；负数给出明确检查提示，0 自动标记已用完，也可在同一表单中切换已用完状态。
- 修改 C 编号后重新查询 EDA 模型，并将新模型属性作为可逐字段采用的建议；不强制覆盖用户维护的库存信息。
- 编辑产生重复身份时并列展示现有记录和待保存记录，只有用户确认后才合并；保存通过记录 revision 防止覆盖其他电脑上的较新修改。
- 删除前重新读取最新记录，并显示名称、编号和库存数量进行不可撤销确认。
- 将有官方库引用的库存元器件导入个人库，并对个人库中的已有器件进行幂等识别；复制返回后必须从个人库回读验证，不再把收藏库当作可写目标或报告未经验证的成功。
- 使用分类化库存总览管理大量元器件：顶部主搜索框覆盖 C 编号、供应商编号、厂家型号、名称、制造商和封装；左侧明确区分系统分类与两级用户分类，器件可从整行拖到分类，拖动已勾选行时可批量移动全部勾选项。
- 通过显式命令幂等导入个人库或收藏库的 EDA 两级分类，不自动移动库存条目；分类树接口不可用时从库内器件已经使用的分类降级汇总，并提示空分类可能遗漏。
- 搜索精确匹配优先于前缀和包含匹配，同等匹配时有库存优先；搜索属于总览本身，不作为菜单项或虚拟元器件显示。
- 在原理图中选择在库元器件并绑定到鼠标放置。
- 批量导入立创商城订单详情 `.xls`/`.xlsx`，在一个多字段窗口中一次设置默认状态和重复策略，预览逐文件订单号、明细、数量和预计库存变化，并显示解析、模型匹配和写入进度；订单号与 SHA-256 文件指纹用于阻止重复导入，同时保留 CSV/JSON 兼容。每次最多选择 100 个文件，单文件上限为 10 MiB；工作簿最多读取 32 个工作表、每表 128 列和 10,000 行数据。
- 导出版本化 JSON 备份；保存目的地与覆盖确认由 EDA 系统保存对话框处理。
- 简体中文和英文界面及菜单。

## 同步与服务器

本扩展没有自建后端，也不要求用户配置数据库。库存使用 `SYS_Storage.setExtensionUserConfig()` 保存。

官方文档将其描述为“扩展用户配置”，但没有承诺跨电脑同步、容量、传播延迟或并发冲突规则。因此仓库当前把跨电脑同步标为 **待 V3 双设备验证**，不把它宣传为已经得到官方保证的云数据库。验证步骤见 [跨设备验证清单](docs/cross-device-validation.md)。在验证完成前，桌面端应配置自动备份，Web 端应定期使用“导出库存备份”。

GitHub 只能托管源码、CI 构建和 Release 文件，不能直接充当运行时数据库。

## 安装

1. 从 Release 或 CI 构建产物中取得 `jlceda-inventory_v0.4.12.eext`。
2. 打开嘉立创 EDA 专业版 V3。
3. 进入“高级 -> 扩展管理器 -> 导入”，选择 `.eext` 文件。
4. 在“已安装”中启用扩展；如需固定在最上方，勾选“显示在顶部菜单”。
5. 从顶部“元器件库存”分组菜单进入“库存总览”或其他功能；扩展不注册额外的顶层直达命令。

## 当前边界

- 嘉立创目前没有公开的商城订单读取 API，因此订单通过立创商城导出的 Excel 订单详情（或自备 CSV/JSON）导入。
- LCSC 官方商品 API 需要申请 key/secret、请求签名和 IP 授权，商城网页也不允许扩展跨域读取。因此无服务器版本只在 EDA 模型未命中或查询失败时打开官方商城供用户确认，不抓取网页，也不冒充已经自动读取商城数据。
- `LIB_Device`、`LIB_Device.copy()`、库分类读取、文件选择和鼠标放置属于官方 BETA API，已集中隔离在平台层，仍需在浏览器版和客户端实测；分类树接口被移除时只会降级导入器件已经使用的分类。
- SCH/PCB 器件读取、选中器件读取、文件系统路径写入和多个新增 IFrame 同样依赖官方 BETA API。自动化测试验证了能力检测、降级、协议和关闭竞态，但不能替代 Web 与桌面客户端实机测试。
- 项目需求当前按快照逐份累计，尚不能识别同一物理板对应的 SCH 与 PCB。若同时保留两份快照会重复计算需求并影响采购建议，当前需用户只保留其中一份或删除重复快照。
- 统一新增表单和库存总览使用官方 IFrame、扩展用户配置和定时器接口；请求、操作和响应均带会话标识，完成后立即清理临时数据。IFrame API 仍处于 BETA，启动失败会明确报错，不会静默切换成行为不同的逐项输入流程，发布前必须在 Web 和桌面客户端实测。
- 鼠标放置成功仅表示元器件已绑定到鼠标，不表示用户完成放置，因此不会自动扣减库存。
- `SYS_Storage` 没有宿主级原子 CAS；插件可阻止同一运行实例内的陈旧写入，但不能承诺多个扩展运行实例或多设备同时写入时的强一致性。
