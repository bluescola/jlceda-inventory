# 库存总览、分类管理与子窗口边界（已确认）

> 状态：已在 `0.4.0` 实现，自动化测试已通过；仍需在嘉立创 EDA V3 Web 端和桌面端完成焦点、滚动与窗口尺寸实机验证。

本次修订覆盖旧版中“分类管理直接展开在分类树下方”和“返回意图后关闭、再重开总览”的交互。数据模型、两级分类限制和搜索规则保持不变；批量删除自 `0.4.13` 起按明确勾选、二次确认和原子 revision 校验受控开放。

## 目标理解

本次调整在一个完整的库存总览中解决浏览与搜索问题：

1. “搜索库存”相当于商城页面的主搜索能力，必须作为库存总览顶部显著的搜索框，不是独立菜单命令，也不得伪装成一条可管理的元器件记录。
2. “库存总览”应从原生下拉选择框升级为适合大量元器件的分类管理界面。交互信息架构可以参考官方商城的分类浏览方式，但不复制商城品牌视觉、分类数据或网页实现。
3. 库存总览是常驻父窗口。打开分类管理、查看详情、完整编辑、重新匹配 EDA 模型、删除确认或其他单条操作时，总览窗口不关闭、不重新创建，背景内容和当前浏览状态继续保留。
4. “管理分类”打开独立的模态管理窗口，参考 EDA 分类管理的双栏结构：左侧一级分类，右侧显示所选一级分类下的二级分类；新增、重命名、排序和删除均在该窗口完成。

目标入口如下：

```text
元器件库存
  - 库存总览
  - 按立创 C 编号添加
  - 添加自定义元器件
  - ...
```

菜单中只有“库存总览”入口，不新增“搜索库存”菜单项。总览表格中的每一行都必须对应真实的 `InventoryItem`。

## 当前实现

- overview 使用请求、操作、响应三通道长会话协议；普通操作返回最新快照，只有用户显式关闭时才结束会话。
- overview 不再使用宿主背景灰色蒙层。IFrame 失焦后短暂等待；只有该失焦与 EDA 主窗口 `focus` 事件配对，且没有正在执行的宿主操作时，才隐藏固定库存总览 IFrame。隐藏不结束长会话，再次调用原“库存总览”菜单命令通过 `showIFrame` 恢复同一会话。
- 总览首次渲染后等待 300 ms，让 EDA 宿主先完成焦点收敛，再显式聚焦 IFrame 内的主搜索框并启用自动隐藏。启用前的启动期 `blur` 必须忽略；稳定聚焦后，用户不先操作分类、详情或表格控件而直接点击 EDA 主窗口时仍能收到第一次有效 `blur`。
- 分类管理、详情和尚未提交的确认弹层不阻止自动隐藏，会随总览一起隐藏；已经提交的导入、复制、移动或其他写入操作完成前阻止隐藏。自动隐藏依赖 IFrame 失焦与 EDA 主窗口聚焦配对，不注册新菜单、不修改 `headerMenus`，也不调用 EDA 左、右、底部面板控制接口。
- 分类管理、详情、完整编辑、删除确认、重复比对和 EDA 模型重匹配全部位于同一 overview DOM 的上层模态，不依赖独立 IFrame 的窗口层级。
- 操作按 `operationId` 串行去重；库存与分类写入校验 revision，会话关闭后丢弃晚到响应。
- 快照更新保留有效的搜索、分类、筛选、排序、分页、滚动和选择状态。

## 入口职责

### 库存总览

- 默认进入分类浏览模式，左侧显示分类树，右侧显示当前分类的库存表格。
- 顶部固定显示主搜索框；搜索框在视觉层级和可操作性上明显区别于库存条目，并支持立即输入查询。
- 支持在表格工具栏中进一步过滤当前结果，但搜索和过滤控件都不是库存条目。
- 用于查看详情、完整编辑、调整分类、删除记录和执行已有的单条库存操作。
- 打开任何子流程时保持总览挂载；子流程完成或取消后回到同一位置，不经过关闭并重开总览的可见过程。
- 首页、原理图和 PCB 菜单均保留该入口。

### 搜索框

- 搜索框属于库存总览本身，作用类似商城列表页的主搜索框。
- 主搜索默认覆盖全部分类，分类树和范围控件用于进一步缩小结果。
- 搜索结果仍可打开详情或进入已有编辑流程，但搜索本身不产生库存变更。
- 已应用搜索存在时，用户清空搜索框会立即恢复当前范围内的全部结果，无需再点击“搜索”或“清空搜索”。
- 不注册 `searchInventory` 或类似的独立菜单命令。
- “从库存放置元器件”继续保持原理图专属入口，不合并进总览搜索。

分类浏览、搜索结果和过滤结果在同一个界面、协议和搜索引擎中完成，不维护第二套库存列表。

## 窗口生命周期与层级

### 常驻总览

- 用户只有显式关闭库存总览时，才结束总览会话并销毁窗口。
- 点击 EDA 主窗口触发的自动隐藏不等于关闭：搜索、分类、筛选、排序、分页、滚动和选择状态继续保留；用户从原菜单再次打开时直接重显已有窗口。
- 用户点击 EDA 原生最小化按钮时，`buttonCallbackFn` 发出的窗口控制通知会短时阻止失焦自动隐藏；`minimizeStyle` 使用 `constricted`，保留可直接点击恢复的缩窄标题窗。V3 没有公开的程序化最小化接口，因此点击 EDA 主窗口的自动行为仍是隐藏而不是伪造宿主折叠动作。
- 如果插件控制器在 EDA 保留隐藏 IFrame 期间被重新创建，旧窗口的操作回调已经不再可信。此时原菜单入口先隐藏并关闭遗留 IFrame，等待宿主窗口收敛后再创建新会话；固定 bridge 的最终清理必须校验 `requestId`，旧控制器不得删除新会话的数据或关闭新窗口，也不得把新请求写给不会重新读取请求的旧页面。
- 活动总览轮询每次先核对 bridge 中的请求所有权；发现新的 `requestId` 或请求已移除时立即结束。总览 ready 后若 EDA 扩展存储上下文因停用、重载或版本切换而不可读，也视为宿主终止会话并静默收尾，不向用户显示业务错误。
- 分类管理、查看详情、完整编辑、重新匹配 EDA 模型、删除确认、重复项比对和错误提示都属于总览之上的子流程，不得把总览关闭作为打开子流程的前置步骤。
- 子流程打开时，总览保持可见但进入不可交互状态，并使用遮罩、焦点约束和明确标题表达当前层级；关闭子流程后焦点返回原触发按钮或对应库存行。
- 所有子窗口都必须位于库存总览的视觉和焦点层级之上。重新聚焦总览时不得把尚未结束的子窗口压到下方；存在活动子窗口时，对总览的点击只应把对应子窗口带到前方或保持背景不可交互。
- 商城跳转在外部窗口打开，总览继续保留；复制常用库和刷新在当前会话中显示进行中状态，结束后局部刷新数据。
- 删除库存成功后只移除对应行并重新计算当前页；分类或筛选导致当前页不存在时，将页码收敛到最后一个有效页，不重置其他状态。

总览必须保留以下状态：

- 搜索文本、搜索范围和输入焦点上下文；
- 当前分类、库存状态、模型状态和排序方式；
- 页大小、当前有效页和表格滚动位置；
- 分类侧栏的展开状态、分类管理窗口中的当前一级分类；
- 仍然有效的多选条目。已删除或移出当前结果的条目从选择集合中移除。

### 子流程承载方式

- 分类管理固定使用库存总览 IFrame 内的模态窗口，不另开宿主 IFrame。
- 查看详情和完整编辑优先使用总览 IFrame 内的模态承载层，并复用现有 `InventoryItemPanel` 的详情、校验和编辑模块，避免复制业务规则。
- 只有在嘉立创 EDA V3 Web 和桌面端均验证支持两个独立业务 IFrame 同时存在、焦点和关闭顺序可靠后，才允许把详情或编辑实现为并行子 IFrame。
- 如果宿主不支持并行 IFrame，必须回退到同一总览 IFrame 内的模态层；不得回退为“关闭总览、打开子窗口、完成后重开总览”。
- 无论内部采用哪种承载方式，用户可见行为、协议结果和验收条件必须一致。

### 重新匹配 EDA 模型

- 点击“重新匹配 EDA 模型”后，在库存总览上方打开专用子窗口；总览留在背景且不可交互，不得先关闭总览，也不得让匹配窗口出现在总览下方。
- 子窗口至少显示当前元器件名称或 C 编号，以及“正在查询”“匹配成功”“系统库无模型”“查询失败”四类明确状态。
- 查询期间禁用重复提交，但允许用户取消等待；取消只关闭匹配子窗口，不修改当前库存记录。
- 匹配成功后先在子窗口展示匹配结果摘要，由用户确认后才关联 EDA 模型；确认成功时关闭子窗口并局部刷新总览对应行。
- 系统库无模型或查询失败时保留结果与重试入口，用户关闭子窗口后回到原总览位置；不得因为失败关闭库存总览。
- 子窗口关闭后焦点返回触发“重新匹配 EDA 模型”的按钮。搜索、分类、筛选、排序、分页、滚动和仍有效的多选状态保持不变。
- 优先在总览 IFrame 内使用最高业务层级的模态窗口实现，从结构上避免宿主窗口层级丢失。如果采用独立子 IFrame，必须在 Web 和桌面端验证 `z-order`、焦点恢复和总览重新聚焦行为；验证失败时必须使用同窗模态实现。

## 总览信息架构

使用独立 IFrame，而不是继续扩展原生 `showSelectDialog`。目标布局为：

```text
+------------------------------------------------------------------+
| 库存总览       [     搜索 C编号 / 型号 / 名称...     ][搜索]   |
|                [库存状态] [模型状态] [范围] [排序] [刷新]       |
+-------------------+----------------------------------------------+
| 全部          128 | 名称 | C编号/厂家型号 | 库存 | 仓位 | 更新时间 |
| 未分类         34 |----------------------------------------------|
| 无源器件          | ...真实库存记录...                            |
|   电阻         21 |                                              |
|   电容         18 |                                              |
| 半导体器件        |                                              |
|   集成电路     12 |                                              |
| [管理分类]        |                                              |
+-------------------+----------------------------------------------+
```

### 分类浏览区

- 分类侧栏明确分为“系统分类”和“用户分类”；“全部”和“未分类”位于系统分类区，是始终存在且不可重命名、移动或删除的虚拟节点。
- 用户分类最多支持两级：一级分类和二级分类；无限层级不属于本文边界。
- 分类名称在同一父级下不得重复，忽略前后空白和大小写差异。
- 分类旁显示条目数，不显示库存数量总和，避免“数量未知”和估算数量造成误导。
- 一级分类展开后，其二级分类区域使用克制的浅底色与一级分类作层级区分；底色只表达从属关系，不使用高饱和色块，也不能削弱当前选中项的识别度。
- 分类浏览区只用于选择分类和显示条目数，不在树节点后堆叠新增、重命名、排序和删除按钮。
- “管理分类”是分类浏览区底部的独立命令，打开双栏分类管理窗口；关闭管理窗口后仍停留在原总览。
- “导入 EDA 分类”只位于双栏分类管理窗口中，不注册 EDA 顶栏命令；导入个人库或收藏库的两级分类后保持管理窗口打开并刷新分类，不自动移动库存条目。
- 不复制官方商城的完整类目表，也不宣称与官方类目同步。

### 双栏分类管理窗口

目标布局参考 EDA 的分类管理信息结构，不复制品牌视觉：

```text
+----------------------------------------------------------------+
| 管理分类                                                    [×] |
+-------------------------------+--------------------------------+
| 一级分类                      | 二级分类                         |
| [+] [重命名] [删除] [↑] [↓]  | [+] [重命名] [删除] [↑] [↓]    |
|-------------------------------|--------------------------------|
| 无源器件                      | 电阻                             |
| 半导体器件                    | 电容                             |
| 连接器                        | 电感                             |
|                               |                                  |
+-------------------------------+--------------------------------+
|                                                    [关闭]       |
+----------------------------------------------------------------+
```

交互边界：

- 左栏只显示用户创建的一级分类；选中一级分类后，右栏只显示其直接二级分类。右栏没有选定一级分类时显示空状态，并禁用“新增二级分类”。
- “全部”和“未分类”继续存在于总览分类浏览区，但不属于用户分类，不进入分类管理窗口，也不能重命名、排序或删除。
- 新增一级分类在左栏创建临时编辑行；新增二级分类在右栏创建临时编辑行。按 Enter 提交，按 Escape 取消，提交失败时保留编辑状态并显示字段级错误。
- 重命名只编辑当前选中项，不弹出第三层窗口。名称保存后保持该项选中；取消时恢复原名称。
- 名称去除前后空白后不能为空；同一父级内忽略大小写后不得重名；允许不同一级分类拥有同名二级分类。
- 上移和下移只调整同栏同级分类的顺序。一级分类不能变成二级分类，二级分类第一阶段不能跨一级分类移动。
- 删除二级分类时，确认窗口显示分类名和受影响库存条目数；确认后删除分类并把其中条目移入“未分类”，不删除库存。
- 删除含二级分类的一级分类时，确认窗口同时显示二级分类数量和整条分支受影响的库存条目数；确认后删除该分支，所有关联条目移入“未分类”。
- 新增、重命名、排序和删除成功后，分类管理窗口保持打开并刷新左右两栏；失败时保持窗口与选择状态，显示可理解的错误，不关闭总览。
- 关闭分类管理窗口时，若当前总览分类仍存在则继续选中；若该分类已被删除，则回到“未分类”，并保留搜索、筛选、排序和页大小。
- 桌面端和 Web 端均支持鼠标与键盘操作。Tab 焦点不离开模态窗口，Escape 先取消行内编辑，再关闭分类管理窗口。

### 库存表格

默认列：

| 列 | 内容 |
| --- | --- |
| 名称 | 库存记录名称，作为主要识别信息 |
| 编号 | 优先显示 C 编号，否则显示厂家型号 |
| 封装 | 优先显示用户维护的库存封装，并保留 EDA 模型封装作为独立信息 |
| 分类 | 当前用户分类；没有分类时显示“未分类” |
| 库存 | 精确数量、估算数量、数量未知或已用完 |
| 最低库存 | 单条目显式阈值；未设置时显示空值 |
| 补货状态 | 充足、低库存、可能不足、需盘点、已用完或未配置 |
| 仓位 | 结构化柜/盒/行/列与原自由文本位置，二者都保留 |
| EDA 模型 | 可用、缺失、查询失败或未查询 |
| 更新时间 | 使用当前本地化数字时间格式 |
| 操作 | 查看详情及其他已有单条操作 |

表格行为：

- 空搜索下的“相关性”当前临时按“有库存优先、全局名称升序”显示，不按分类分组；只有显式选择“分类顺序”时才按用户分类分组。该语义尚未最终确认，后续决策问题和验收样例见[库存总览“相关性”排序待决策记录](relevance-sorting-decision.md)。
- 支持按名称升序、库存数量降序、更新时间倒序和用户分类顺序排序。
- 单击名称或查看操作在总览上方打开只读完整详情；详情覆盖 C 编号、供应商编号、身份、库存、分类、商城、EDA 模型、来源、时间和版本字段，C 编号独立显示并可复制。库存封装与 EDA 模型封装分别显示，两者都不存在时统一显示“—”。完整编辑在同一子流程层级打开，二者都不得关闭总览。
- 详情进入完整编辑后，保存成功刷新对应行并关闭子流程；取消编辑返回详情，关闭详情才回到总览。
- 完整编辑保存失败、C 编号重新匹配或重复项比对期间，总览保持挂载且不可在背景触发第二个写操作。
- 支持多选后批量移动分类。
- 勾选或取消勾选触发列表重绘时必须恢复表格的横向与纵向滚动位置，当前所见行不得跳回顶部；筛选、切换分类和翻页仍按新结果重置浏览位置。
- 批量删除只作用于用户明确勾选的记录，提交前必须显示选中数量并进行不可撤销确认；服务端先校验全部条目 revision，再通过一次持久化原子删除，任一条目缺失或过期时整批不写入。仍不提供批量改数量或批量修改 C 编号。
- 数据量达到 1000 条以上时仍不得一次创建全部 DOM 行；采用分页或虚拟列表，每页建议 50 条。

### 分类分配与快捷整理

- 元器件通过 Pointer Events 从名称前的拖动手柄或表格行非交互区域拖到分类浏览区，可以放入“未分类”、任意一级分类或任意二级分类；拖动已勾选的记录时移动全部勾选项。只有至少一条记录需要改变分类时才提交，放回原分类或取消拖动不得写库存。
- 拖动优先使用指针捕获；宿主不支持指针捕获、捕获失败或捕获中途丢失时，必须使用 `document` 级 `pointermove`、`pointerup` 和 `pointercancel` 监听继续并结束当前拖动，不能让指针离开拖动柄后丢失事件。
- 批量移动工具固定提供横向排列的“一级分类”和“二级分类”两个字段。一级分类未选择时禁用二级分类；一级分类没有二级分类时显示明确空状态。用户可以只选择一级分类，也可以选择其下的二级分类，最终提交实际目标分类 ID。
- 快捷分类采用明确的批量操作：用户先应用搜索或其他筛选，常见入口是筛选“未分类”，再执行“全选当前筛选结果”，最后通过上述两个分类字段批量移动。这里的全选覆盖当前完整筛选结果，不只覆盖当前分页中可见的行，并在提交前显示选中数量。
- “删除所选”复用同一组选中结果，不默认扩大选择范围；确认弹层必须显示实际删除数量，取消、并发冲突或协议失败均不得删除任何记录。
- 扩展不得根据名称、C 编号、制造商、封装、EDA 模型或历史操作自动猜测并写入分类；快捷分类只是缩短筛选、全选和批量移动的操作路径，最终分类始终由用户明确选择。

### 过滤器

- 分类：来自左侧分类树。
- 库存状态：全部、有库存、已用完，默认“全部”。
- EDA 模型状态：全部、可用、缺失、查询失败、未查询。
- 补货状态：全部、需要补货、需要盘点。
- 收藏：全部、只看收藏。
- 搜索文本：匹配 C 编号、供应商编号、厂家型号、名称、制造商和封装。
- 清空过滤器后恢复当前模式的默认状态，不修改任何库存数据。

## 搜索规则

搜索用于定位库存，不再表达“替代料推荐”或“采购建议”。排序边界为：

1. 完全匹配优先于前缀匹配，前缀匹配优先于包含匹配。
2. 匹配等级相同时，有库存记录优先于已用完记录。
3. 其余同分记录按更新时间倒序。
4. 空关键词表示全部库存，包括已用完记录；库存状态必须清楚显示，不得再使用“全部可用库存”文案。
5. 搜索不得查询商城网络、抓取网页或自动创建库存记录。

## 分类数据边界

当前版本使用 schema-v9；分类结构仍保持 schema-v3 引入的两级边界，后续 schema 在同一文档中增加订单关联、补货元数据、BOM 流水、项目采购和替代关系：

```ts
interface InventoryCategory {
	id: string;
	name: string;
	parentId?: string;
	sortOrder: number;
	createdAt: string;
	updatedAt: string;
	revision: number;
}

interface InventoryItem {
	// 既有字段保持不变
	categoryId?: string;
	minimumQuantity?: number;
	favorite?: boolean;
	datasheetUrl?: string;
	structuredLocation?: StructuredInventoryLocation;
}

interface InventoryDocument {
	schemaVersion: 9;
	categories: InventoryCategory[];
	items: InventoryItem[];
	orderImportBatches: OrderImportBatchRecord[];
	transactions: InventoryTransaction[];
	stockOutBatches: StockOutBatchRecord[];
	projectSnapshots: ProjectDemandSnapshot[];
	purchaseRecords: PurchaseRecord[];
	substituteLinks: InventorySubstituteLink[];
}
```

约束如下：

- 分类属于用户库存组织数据，不属于 `PartIdentity`，不得参与重复元器件身份判断。
- schema-v1 至 v8 数据迁移到 schema-v9 时，所有既有条目保持原 ID、数量、状态、revision 和时间；缺少的新集合初始化为空，不根据名称或封装自动创建替代关系。
- 迁移不得根据名称、封装或厂家自动猜测分类。
- 新增元器件只有在调用方显式传入有效 `categoryId` 时进入指定分类；现有总览外添加与导入命令默认进入“未分类”。
- 移动分类和批量删除都必须经过应用服务并校验条目 revision，IFrame 不直接写仓储；批量删除在全部校验通过后只保存一次。
- 订单导入文件只保存规范化物料身份键与库存 ID 的关联，不保存原始订单行；删除库存后关联自动失效，重新导入只恢复缺失物料，未删除物料不得再次累加。
- 分类 CRUD 必须经过 `InventoryService` 的独立应用层操作，并检查分类或条目的 expected revision；每次保存继续推进文档 revision。
- `SYS_Storage` 不提供原子 CAS，revision 校验只能防止同一已加载快照内的陈旧覆盖，不能承诺跨设备强一致性。
- 继续使用现有 `inventory.v1.document` 存储键和扩展 UUID，不另建第二份库存主数据。

## UI 与协议边界

- 保留现有 `InventoryOverviewPanel` 端口，继续负责包含分类与搜索能力的库存总览。
- 保留独立、版本化的 overview IFrame 协议，不扩充单条 `InventoryItemPanel` 协议承担列表职责；本轮为长会话行为升级协议版本。
- overview 协议是长生命周期会话。主运行时向 IFrame 提供只读快照、分类和本地化标签；IFrame 发送带操作 ID 的 `view-item`、`edit-item`、`delete-item`、`delete-items`、`open-marketplace`、`open-datasheet`、`retry-model`、`copy-common`、`move-items`、`export-replenishment`、分类 CRUD、排序和 `refresh` 请求，但这些请求本身不是关闭总览的终态结果。
- `retry-model` 是带阶段响应的子流程：至少返回查询中、待确认、成功、未命中、失败和取消状态。只有用户确认且应用服务保存成功后，最新模型状态才写回总览快照。
- 控制器处理请求后向同一会话返回成功、取消、失败以及最新快照或受影响数据；IFrame 根据响应关闭子流程或保留错误状态。只有用户显式关闭总览才写入会话终态并清理桥接键。
- 每个请求携带经过校验的 revision、操作 ID 和必要视图上下文；同一记录或分类存在未完成写操作时禁用重复提交，过期响应不得覆盖较新的界面状态。
- 控制器在同一挂载会话中处理意图并回写响应；不再通过“关闭总览、处理、重开窗口”的循环实现普通操作。
- 所有写操作由控制器调用库存或分类应用服务完成，成功后重新生成快照。
- 窗口通信继续使用请求作用域键、超时、清理和诊断规则，不记录搜索文本、元器件内容或完整仓位。
- 自动隐藏只在库存 IFrame 已稳定聚焦后，其失焦与 EDA 主窗口 `focus` 事件配对时触发；切换到 VS Code 等外部应用只有失焦、没有 EDA 主窗口聚焦，不得隐藏总览。
- Web 和桌面客户端都必须验证窗口尺寸、模态层级、分类双栏滚动、表格滚动、焦点恢复、键盘操作和关闭竞态；重新匹配 EDA 模型窗口还必须验证始终位于总览上方，不会因点击或聚焦总览而落到下层。
- 小尺寸窗口中分类树改为可展开侧栏；不得让分类区、表格或操作按钮互相遮挡。
- 小尺寸窗口中分类管理仍保持“一级/二级”的信息层级，可以把双栏改为两个页签或分步面板，但不得把所有分类重新混成无层级长列表。

## 性能边界

- 每次打开或显式刷新只读取一次库存文档，不为每行重复查询仓储或 EDA 模型。
- 子流程写入成功后只重新读取一次库存文档并更新当前会话，不销毁并重建总览 IFrame。
- 分类计数、过滤和排序基于同一份内存快照完成。
- 输入搜索可在本地防抖，但不得产生网络请求。
- 以 5000 条库存、200 个分类作为首轮容量验证上限；记录打开耗时、过滤响应和内存占用。
- 容量验证结果不达标时先完善分页或虚拟化，不引入后端服务作为补救。

## 明确不在本文范围

- 复制、抓取或实时同步官方商城分类树。
- 根据商品名称、封装或模型自动猜测并写入分类。
- 根据参数、名称或封装自动判断兼容性。用户确认的替代关系由独立命令维护，只做人工候选排序，不属于总览搜索规则。
- 标签、多分类归属、智能文件夹或无限层级分类。
- BOM 校验/出库流水和项目采购使用独立操作窗口，不塞入库存总览；放置后自动扣库存仍不实现。
- 批量修改元器件身份与数量。
- 对跨设备同步作出新的保证。
- 改变商城商品信息与 EDA 模型相互独立的既有边界。

## 验收条件

- 浏览与搜索只有“库存总览”入口，不影响添加、导入、备份和诊断等其他菜单命令。
- 打开分类管理、详情、完整编辑、重新匹配 EDA 模型、删除确认和其他行操作时，库存总览窗口始终可见且不被关闭；完成或取消后恢复原浏览状态与焦点。
- 重新匹配 EDA 模型以总览上方子窗口呈现，覆盖查询、结果确认、未命中、失败、重试和取消状态；在 Web 与桌面端均不会被总览窗口压到下方。
- 总览顶部具有显著搜索框，列表中不再出现值为 `search-inventory` 的伪库存行。
- 总览能按两级分类浏览，并能组合库存状态、模型状态和文本过滤。
- 分类管理使用左侧一级、右侧二级的独立模态窗口；全部分类 CRUD、排序、删除确认和错误处理不关闭分类管理或总览。
- 全选当前筛选结果后可以删除所选记录；二次确认显示数量，成功时整批删除，任一 revision 冲突时整批保留。
- 搜索结果能打开详情和编辑子流程，但搜索动作本身不写数据，子流程关闭后仍保留搜索结果位置。
- 支持的 schema-v1 至 v8 数据迁移到 schema-v9 后，旧条目仍可见于原分类或“未分类”，原有业务字段与 revision 保持其既有语义。
- 新建、重命名、排序、删除分类以及批量移动分类均有单元测试和协议测试。
- 删除非空分类只移动条目，不删除库存。
- overview 长会话协议具有请求关联、并发提交保护、取消、失败、过期响应和显式关闭测试；不得只测试重开后恢复 `viewState`。
- 5000 条库存下界面仍可滚动、搜索和切换分类，不出现明显输入阻塞。
- 简体中文和英文标签同步，Web 与桌面端完成实机验证。

## 已确认决策

实现遵循以下方案：

1. 首页、原理图和 PCB 的浏览与搜索只使用“库存总览”入口；搜索是总览顶部的主搜索框，其他业务菜单保持独立。
2. 分类浏览、搜索和过滤保留在总览；分类管理使用总览 IFrame 内独立的双栏模态窗口，不再展开在分类树底部。
3. 总览是常驻父窗口，详情、编辑、确认和分类管理均不得以关闭总览为代价；并行宿主 IFrame 不是交互成立的必要条件。
4. 分类由用户本地维护，最多两级；左栏一级、右栏二级，不复制官方商城的实际分类数据。
5. “全部”和“未分类”为不可删除的系统节点，旧数据统一迁入“未分类”。
6. 删除一级分类会删除其二级分类分支，删除任何分类都只把关联条目移入“未分类”，不删除库存。
7. 分类只支持同级排序；二级分类第一阶段不支持跨一级分类移动。
8. 支持批量移动分类和经二次确认的原子批量删除；仍不支持批量修改身份、C 编号或数量。
9. 空搜索显示包含已用完记录的全部库存，并提供独立库存状态过滤器。
10. 使用上文列出的默认表格列和 5000 条容量目标。
11. 重新匹配 EDA 模型使用位于总览上方的子窗口；匹配结束、失败或取消都不关闭总览，并保留原浏览状态。
12. 实现可以分阶段推进，但必须完成长会话协议、常驻总览、双栏分类管理、详情、编辑与模型重匹配子流程、响应式界面和完整验证后再交付用户测试。

以上边界作为下一轮实现和最终验收依据。
