# EDA 分类导入与菜单边界

## 分类导入

“导入 EDA 分类”是用户显式触发的只读导入：扩展读取当前工作区的个人库或收藏库分类，并把一级、二级分类幂等地添加到库存分类中。同级同名分类会合并，只新增缺少的分类；已有分类不会重命名或删除，库存条目也不会被自动移动。EDA 返回的虚拟 `ALL` 根节点不会导入用户分类，其中包含的实际分类会提升到正常层级。

首选接口是 [`LIB_Classification.getAllClassificationTree()`](https://prodocs.lceda.cn/cn/api/reference/pro-api.lib_classification.getallclassificationtree.html)。官方将它标记为 BETA，本地官方类型定义进一步注明它从 EDA v3.2 起弃用、在 v3.3 移除。因此实现会在该接口不可用时分页搜索库内器件，并从器件返回的 `classification: string[]` 汇总已使用的分类路径。

降级读取不是完整分类同步：没有被任何器件使用的空分类无法发现；旧版返回 UUID 分类索引而非名称数组时也无法恢复名称。界面会明确提示本次结果不完整。两个读取路径都不可用时不写库存，并显示当前 EDA 版本不支持。

## 菜单边界

`extension.json` 的 `headerMenus` 逐项保持 0.4.3 的原始配置：只在首页、原理图和 PCB 三个既有上下文注册“元器件库存”分组，不新增“库存总览”直达命令，也不把“导入 EDA 分类”挂到 EDA 顶栏。分类导入的唯一可见入口位于库存总览的“管理分类”子窗口中；导入完成后该窗口保持打开并用最新分类快照刷新左右两栏。清单测试会逐项锁定原菜单，避免入口改动导致宿主丢弃整个扩展菜单。

官方扩展清单没有文档化“仅在本地安装成功后触发”的激活事件；当前官方 SDK 模板仍使用空的 `activationEvents`。因此扩展不在 `activate()` 中自动弹出窗口，避免把普通启动或重新激活误判为刚安装。用户可在扩展管理器中启用扩展并勾选“显示在顶部菜单”，但导入后菜单何时刷新仍由 EDA 宿主管理，必要时需切换编辑页面或重启 EDA。
