# 0.5.9 新功能手动测试指南

本文只写 `0.5.0` 至 `0.5.9` 新功能的手动测试操作。F0-F9 可以逐项单独测试，不要求一次全部完成。

## 开始前

1. 会修改或扣减库存的测试先执行 `元器件库存 -> 另存库存备份`，或只使用可删除的临时记录。
2. 每节开始前关闭同类旧面板，在库存总览点击顶部的“清除筛选”。该按钮会同时清除搜索、分类、库存、补货、收藏和 EDA 模型筛选。
3. 文中的 `<C_A>`、`<C_B>` 等必须替换成实际 C 编号。同一测试内 A、B、C、D、M、X 的 C 编号要互不相同，且库存中不能有其他同 C 编号记录。
4. `F0-已用完`、`F0-无模型` 允许不填 C 编号，因为它们只用于验证不可放置状态；其余样本使用有效 C 编号。

## 这些状态怎样建立

以下名称中，只有“数量类型”是表单选项；模型状态和补货状态都是保存后由插件计算出来的，不能在表单里直接选择。

| 要建立的状态 | 实际操作 |
| --- | --- |
| 真实数量 | 新增表单的“数量类型”选择“真实数量”，再填写数量 |
| 大概数量 | 新增表单的“数量类型”选择“大概数量”，再填写数量 |
| 数量未知 | 只能在新增表单选择“数量未知”；数量输入框随后禁用 |
| 已用完 | 新增表单选择“已用完”，或在“完整编辑”勾选已用完 |
| EDA 模型可用 | 新增表单填写 C 编号后点击“查询 EDA 器件模型”，看到已找到模型，再保存 |
| 模型缺失 | 点击“查询 EDA 器件模型”，看到系统库暂无对应模型，再保存 |
| 尚未查询 | 不点击“查询 EDA 器件模型”，直接保存 |
| 低于最低库存 | 真实数量小于最低库存，例如数量 `1`、最低库存 `3` |
| 可能需要补货 | 大概数量小于最低库存，例如大概 `2`、最低库存 `3` |
| 需要盘点 | 新增时同时选择“数量未知”并填写最低库存，例如最低库存 `3` |

新增表单的“名称”为必填项。通过 C 编号查询到模型后通常会自动带出名称；没有带出时要手工填写。

当前“完整编辑”只有“真实数量”和“大概数量”，没有“数量未知”。因此未知数量记录必须在新增时一次填好最低库存，后续不要用“完整编辑”保存它。这是当前已确认的功能缺口，不是测试者漏看了选项。

## 公共测试数据

每节只准备自己会用到的记录，不需要一次建齐。

| 代号 | 新增表单填写内容 | 其他要求 |
| --- | --- | --- |
| A | 名称 `测试-A`，真实数量 `10` | F0 前查询并绑定模型；放入分类“手动测试” |
| B | 名称 `测试-B`，真实数量 `1` | 最低库存先留空 |
| C | 名称 `测试-C`，大概数量 `2` | 最低库存先留空 |
| D | 名称 `测试-D`，数量未知，最低库存 `3` | 必须在新增时一次建好 |
| S | 名称 `测试-S`，真实数量 `6` | 只用于 F9 替代料 |
| M | 不创建库存记录 | EDA 画布或外部 BOM 中存在该 C 编号 |
| X | 不要求库存记录 | 使用与 A-D 不同的有效 C 编号和 EDA 模型，并把 EDA 原生“加入BOM”设为“否” |

第一次准备 A-D 时按以下操作执行：

1. 打开库存总览，点击左上角分类图标并进入“管理分类”。“手动测试”不存在时，点击一级分类栏的 `+` 新建；已经存在时直接复用。新增表单只能选择已有分类，不能在表单中临时创建。
2. 在 EDA 自带器件库中选取 A、B、C、D 四个不同模型，记下各自 C 编号。对每个编号都先在库存总览把搜索范围选为“全部分类”，输入完整 C 编号并点击搜索：首次准备只使用结果为 `0` 条的编号；重测时若恰好已有一条状态正确的测试记录，直接复用，不要重复新增。
3. 只对尚未建立的记录执行 `元器件库存 -> 按立创 C 编号添加`，填写 C 编号和上表内容。A 必须点击“查询 EDA 器件模型”并确认查询成功，再选择一级分类“手动测试”；B、C、D 不要求绑定模型。
4. 保存后先在库存总览 A 所在行确认“分类”列为“手动测试”，再逐条点击“查看详情”：A 的数量为 `10`、数量类型为“真实数量”、EDA 模型为可用；B 的数量为 `1`、数量类型为“真实数量”；C 的数量为 `2`、数量类型为“大概数量”；D 的数量和数量类型均为“数量未知”，最低库存为 `3`。

## F0：从库存总览放置元器件

入口：原理图的 `元器件库存 -> 从库存放置元器件`

### 前置条件

1. 打开可编辑的 SCH；A 为真实数量 `10`、模型可用、分类为“手动测试”。
2. 执行 `添加自定义元器件`，填写名称 `F0-已用完`，C 编号留空，数量类型选择“已用完”后保存。
3. 再执行 `添加自定义元器件`，填写名称 `F0-无模型`，C 编号留空，选择“真实数量”并填 `1`，不要查询模型，直接保存。
4. 关闭已经打开的库存总览。

### 测试步骤

1. 执行入口。新打开的总览应默认只显示“有库存 + 模型可用”的记录，不出现逐条原生单选列表。
2. 先在搜索框输入 A 的完整 C 编号并点击搜索，确认能定位 A；随后点击“清空搜索”，再打开分类栏选择“手动测试”，重新确认 A 可见。搜索和分类必须分开验证。
3. 不要点击 A 的名称；直接在 A 所在行的“操作”列点击第一个“放置”按钮。总览应自动隐藏，器件绑定到鼠标；在 SCH 画布点击一次完成放置。
4. 执行普通的 `元器件库存 -> 库存总览` 并点击“清除筛选”。`F0-已用完` 和 `F0-无模型` 应可见；两行的“放置”按钮均禁用，悬停按钮可分别看到库存已用完或没有可用 EDA 模型的原因。
5. 核对 A 和两个负向样本的库存数量都没有变化。

通过标准：可在总览中搜索、按分类选择并放置；成功后面板不挡住画布；不可放置项给出原因；放置不自动扣库存。

收尾：删除 `F0-已用完` 和 `F0-无模型`。

## P0：自动备份与恢复

入口：`另存库存备份 / 自动备份设置 / 恢复库存备份`

### 前置条件

A=`10`、B=`1`；准备两个没有重要文件的可写测试目录，记为“目录一”和“目录二”。记录自动备份当前是否启用及当前目录。原生目录选择只在已开启“允许外部交互”的桌面端可用。

### 测试步骤

1. 执行“另存库存备份”，保存 `p0-original.json`。
2. 在库存总览搜索 A，执行“完整编辑”，把数量改成 `7`。恢复 `p0-original.json`，A 应回到 `10`。
3. 再执行“恢复库存备份”，选择“上次恢复前快照”，A 应回到 `7`；随后把 A 手工恢复为 `10`。
4. 只有从未配置过自动备份时才测首次分支：打开“自动备份设置”应直接出现 EDA 原生目录选择器。先取消，设置不变且不报错。已经配置过的账号把此步记为“不适用”。
5. 再打开设置并选择目录一。目录中应自动生成 `jlceda-inventory-latest.json`，用户不需要手建 JSON；文件应为 schema-v9。
6. 再打开设置，选择“选择备份目录”。第一次取消时应继续使用目录一；再次打开设置并选择“选择备份目录”，改选目录二后，应在目录二创建固定 JSON。
7. 把 B 从 `1` 改为 `2`，固定 JSON 中的 B 应自动变为 `2`，过程不弹窗。
8. 打开设置并执行“立即测试备份”，应成功且本次设置命令结束。再次打开“自动备份设置”，选择“停用自动备份”；把 B 从 `2` 改为 `3`，固定 JSON 仍应是 `2`。
9. 打开设置并选择“使用当前文件重新启用”，固定 JSON 应立即变为 B=`3`。再把 B 依次改为 `4`、`5`，每次都应更新 B、`revision` 和 `updatedAt`。
10. 保持自动备份启用，执行“另存库存备份”保存当前快照；把 A 改为 `6` 后恢复该快照。恢复完成后重新执行一次“另存库存备份”，将新导出与固定 JSON 比较，库存内容、`revision`、`updatedAt` 应一致。
11. 恢复“上次恢复前快照”，A 应回到 `6`；再次另存当前库存并与固定 JSON 比较，两者也应一致。
12. 临时关闭扩展的“允许外部交互”，再选备份目录，应提示重新授权且不改变原设置。恢复权限后应可继续选择。Web 端只需确认固定目录不受支持，手动另存和恢复仍可用。

通过标准：不填写路径、不手建 JSON；取消无副作用；停用后文件不更新；自动备份启用时，每次库存保存或恢复后固定 JSON 都与当前库存一致。

收尾：恢复 A=`10`、B=`1`。如果测试前已有配置，恢复原目录和启用状态；如果测试前从未配置，当前 UI 无法清除历史目录，只能停用自动备份，测试目录仍会保存在配置中。这是当前已知限制。

实机结论（2026-07-23）：`0.5.5` 桌面端指定用例已通过，包括目录选择与取消、权限拒绝、固定 JSON 创建、停用与重新启用、连续保存、外部文件恢复和恢复前快照恢复。最终固定 JSON 与当前库存导出的 SHA-256 一致。永久写入失败、损坏备份、Web 降级和跨设备行为未包含在该次通过结论中。

## F1：当前设计与外部 BOM 库存校验

两个入口彼此独立：

- SCH/PCB 的 `校验当前设计库存` 读取当前设计，不打开文件选择器。
- 首页、SCH、PCB 的 `校验外部 BOM 库存` 会打开文件选择器，不读取或放置画布器件。

### F1-A：校验当前设计库存

前置条件：A=`真实 10`、B=`真实 1`、C=`大概 2`、D=`数量未知`；四条记录不能有同 C 编号重复库存。

M 的建立方法：从 EDA 自带器件库选择一个插件库存中不存在的有效 C 编号。在库存总览把搜索范围选为“全部分类”，输入完整 C 编号并点击搜索，确认结果为 `0` 条，再把该 EDA 器件放入测试 SCH。不要在插件中为 M 新增库存。找不到这种器件时，记录“M 分支未覆盖”，不要伪造库存状态。

X 的建立方法：从 EDA 自带器件库另选一个与 A-D、M 都不同的模型，记下其 C 编号并放入测试 SCH。确认选中该器件后，右侧属性面板存在 EDA 原生“加入BOM”属性；X 不需要插件库存记录。宿主不提供该属性时，按下方步骤把 X 分支记为未覆盖。

1. 在可编辑测试 SCH 中放置 A 两个，B、C、D、M、X 各一个。A-D 和 X 使用带有效 C 编号的 EDA 模型，画布 C 编号与库存一致；M 按上面的专门方法建立。X 使用与 A-D 不同的器件，不需要在插件中新增库存记录。
2. 选中 X，在 EDA 右侧原生属性面板把“加入BOM”设为“否”。这不是插件表单选项；当前宿主若不显示该原生属性，记录“X 分支因宿主选项不可用而跳过”。
3. 关闭库存总览和旧的库存校验报告，返回放置了测试器件的 SCH 或 PCB，使该设计页面处于当前激活状态。
4. 在当前 SCH/PCB 页面打开 `元器件库存 -> 校验当前设计库存`。该命令不会打开 BOM 文件选择器；如果菜单里只有“校验外部 BOM 库存”而没有本命令，停止本节，记录“当前设计库存校验菜单缺失”并导出诊断日志，不能用外部 BOM 入口替代。
5. 在随后出现的“输入需要生产的板数”中填 `2` 并确定，等待“当前设计库存校验”报告打开。
6. 报告只有“状态筛选”，选择“全部状态”，核对：

   | 样本 | 需求与结果 |
   | --- | --- |
   | A | 总需求 `4`，状态“充足” |
   | B | 总需求 `2`，状态“不足”，缺口 `1` |
   | C | 需求 `2`，需复核 |
   | D | 需求 `2`，需盘点 |
   | M | 需求 `2`，库存缺失 |
   | X | 不出现在报告中 |

7. 逐项切换状态筛选并核对界面，随后恢复“全部状态”再点击“导出缺料 CSV”。导出的 CSV 是完整报告，不会只导出当前筛选结果；核对测试前后库存数量不变。

通过标准：实际建立的每个分支需求和状态正确。未建立 M 或 X 时只能把对应分支记为“未覆盖”，不能记整项完整通过。

实机结论（2026-07-23，`0.5.9`）：F1-A 通过。板数为 `2`；完整报告覆盖充足状态，导出的 `logs/jlceda-inventory-shortage-2026-07-23.csv` 覆盖不足、需复核、需盘点和库存缺失，其中 C10081/R23 为需盘点、C164085/U14 为需复核、C5832371/L1 为不足。画布中的 R22 设置“加入BOM=否”后未进入报告和 CSV，X 排除分支通过；校验与导出未修改库存。

已记录交互问题：库存校验报告窗口当前只能最大化，不能最小化或缩小，会遮挡画布下方器件，不便于边看设计边核对报告。该问题不影响本次判定结果和 CSV 导出，因此不阻断 F1-A 通过结论。

### F1-B：校验外部 BOM 库存

前置条件：A=`真实 10`、B=`真实 1`、C=`大概 2`、D=`数量未知`，四条库存均为同 C 编号唯一记录。另选 `<C_M>`，在库存总览把搜索范围设为“全部分类”，输入完整 C 编号并点击搜索，确认结果为 `0` 条。本节不需要 X、画布器件或 EDA 模型。

建立 `f1-stock-check.csv`：

```csv
C编号,位号,数量,名称
<C_A>,A1 A2,2,测试-A
<C_B>,B1,1,测试-B
<C_C>,C1,1,测试-C
<C_D>,D1,1,测试-D
<C_M>,M1,1,测试-M
,U_TEST,1,
```

最后一行用于“身份未匹配”边界；不测该分支时可以删除。

1. 从首页、SCH 或 PCB 执行“校验外部 BOM 库存”，选择该 CSV。取消文件选择时应直接结束且库存不变。
2. 在映射面板选择唯一的 CSV 工作表，确认表头行，将 C 编号、位号、数量、名称映射到对应字段。
3. 保留最后一行时，映射后会提示解析问题；确认继续，才能在报告中验证“身份未匹配”。
4. 输入板数 `2`，核对 A 充足、B 不足 `1`、C 需复核、D 需盘点、M 库存缺失；保留空 C 行时还应有“身份未匹配”。
5. 逐项切换状态筛选并核对界面，恢复“全部状态”后导出缺料 CSV。CSV 是完整报告，不随当前筛选变化；核对库存数量未变化。

通过标准：选择文件后能完成映射并生成上述结果；不会放置器件或修改库存。

## F2：最低库存与补货筛选

入口：`库存总览 -> 完整编辑`，以及总览顶部“补货状态”

前置条件：A=`真实 10`、B=`真实 1`、C=`大概 2`，三者最低库存留空；D 必须已经按公共数据方法建成“数量未知 + 最低库存 3”。

1. 分别搜索 A、B、C，执行“完整编辑”，把最低库存设为 `3`。不要打开 D 的完整编辑。
2. 点击“清除筛选”，核对状态原文：A“库存充足”、B“低于最低库存”、C“可能需要补货”、D“需要盘点”。
3. 把 A 的数量改为 `0`，或勾选“已用完”，A 应显示已用完。
4. “补货状态”选择“需要补货”，结果中应包含 A、B、C且不包含 D；其他真实库存中的补货记录可以同时存在。
5. “补货状态”选择“需要盘点”，结果中应包含 D 且不包含 A、B、C；其他真实库存中的需盘点记录可以同时存在。
6. 点击“导出补货清单”。导出包含整个库存中的全部补货和盘点项，不随当前总览筛选变化；按 A-D 的 C 编号定位并核对四条状态。

通过标准：最低库存只影响提示和筛选，不会自动改变数量。

收尾：A 恢复为真实数量 `10`；清空 A、B、C 的最低库存。D 无法通过当前完整编辑无损清空最低库存，不再使用时直接删除该临时记录。

## F3：定位选中元器件库存

入口：SCH/PCB 的 `定位选中元器件库存`

前置条件：画布中放置 A、B、M 各一个；A、B 各自匹配唯一库存。M 有有效 C 编号，并已在库存总览把搜索范围选为“全部分类”、输入完整 C 编号并点击搜索，确认结果为 `0` 条。

1. 返回画布并清除选择，只选中 A，执行入口。库存总览应打开、滚动到 A 并短暂高亮。
2. 关闭或隐藏总览，返回画布，清除选择后同时选中 A、B，再执行入口。原生选择器应要求选择要定位的器件；选择 B 后总览应定位 B。
3. 关闭或隐藏总览，返回画布，清除选择后只选 M，再执行入口。应提示没有库存，并询问是否打开预填新增表单。
4. 确认打开新增表单后点击“取消”，库存中不应新增 M。
5. 再回到画布，清除全部选择后执行入口，应提示“当前没有选中可识别的元器件。”

通过标准：每轮定位目标正确；除非最终保存新增表单，否则库存不变。

## F4：EDA 模型候选与人工确认

入口：`库存总览 -> 未绑定模型的记录 -> 重新匹配 EDA 模型`

### 前置条件

1. “手动测试”分类不存在时，先从库存总览的“管理分类”新建该一级分类。随后选择一个有效且库存中未重复的 C 编号，执行“添加自定义元器件”，填写名称 `F4-待匹配`、真实数量 `1`、分类“手动测试”、位置 `F4-LOC`、备注 `F4-NOTE`。不要点击“查询 EDA 器件模型”，直接保存；记录应显示“尚未查询”。不能把 C 编号留空，因为无 C 编号记录不会显示“重新匹配 EDA 模型”。
2. 如果该 C 编号能被 EDA 系统库精确查到，重新匹配时会直接进入最终绑定确认，这是可测试的主链路。如果精确查询未命中，才会继续出现搜索词输入框；为该分支另准备一个能搜到至少一个候选的名称或厂家型号。
3. “模型缺失”状态是条件分支：只有点击“查询 EDA 器件模型”并看到“EDA 系统器件库暂无对应模型”后保存，才会产生该状态。没有这种真实 C 编号时记为未覆盖，不影响测试“尚未查询”记录的绑定确认。

### 测试步骤

1. 在总览找到 `F4-待匹配`，点击“重新匹配 EDA 模型”。若该 C 编号精确命中，会直接进入最终绑定确认；若出现搜索词输入框，先输入一个随机唯一字符串。确认无结果且库存不变后，点击当前弹窗中的“重新匹配 EDA 模型”，再输入准备好的有效搜索词，取得至少一个候选。
2. 出现多个候选时，核对候选行中 EDA 实际返回的非空字段；候选行最多包含名称、C 编号、厂家型号、封装、符号和封装模型，外部数据为空的字段不会显示。只有一个候选时会跳过候选选择器，直接进入最终确认，这不是失败。
3. 在最终绑定确认中点击“取消”，记录应仍未绑定模型。
4. 再次执行重新匹配，取得候选并在最终绑定确认中确定。总览的 EDA 模型状态应变为“模型可用”。
5. 打开详情，确认名称仍为 `F4-待匹配`、数量仍为 `1`、分类仍为“手动测试”、位置仍为 `F4-LOC`、备注仍为 `F4-NOTE`。

通过标准：至少一个候选只在最终人工确认后绑定，取消不修改库存；实际出现搜索词输入框时，无结果也不修改库存。多候选、模型缺失和 API 查询失败按实际条件分别记录覆盖情况。

收尾：删除 `F4-待匹配` 临时记录。

## F5：包装码入库

入口：首页 `扫描包装码入库`

前置条件：选择一个库存中不存在的新有效 C 编号 `<C_F5_A>`，通过“添加自定义元器件”建立临时记录：名称和厂家型号都填 `MT-PACKAGE-A`，真实数量填 `10`，模型不要求绑定。确认它是该 C 编号的唯一库存，并先另存库存备份。

使用以下内容，将 `<C_F5_A>` 替换成实际编号：

```text
{pc:<C_F5_A>,pm:MT-PACKAGE-A,qty:5}
```

1. 执行入口并粘贴包装码，核对预览中的原文、C 编号、厂家型号和数量。点击取消，命令应结束，F5-A 仍为 `10`。
2. 重新执行入口并再次粘贴同一内容。确认预览后只应打开预填新增草稿，库存仍未变化。
3. 在草稿点击“保存到库存”，应出现重复比较。点击“继续编辑”，比较窗口关闭，F5-A 仍为 `10`。
4. 再次点击“保存到库存”，重新出现重复比较；点击“确认合并”，F5-A 应变为 `15`，名称和厂家型号仍为 `MT-PACKAGE-A`。
5. 以下错误样本必须各自重新执行一次“扫描包装码入库”，均应在打开草稿前报错：

   - `pm:ABC,qty:1`：缺少 `pc`
   - `pc:C123,qty:0`：数量不是正整数
   - `pc:C123,pc:C456,qty:1`：重复字段冲突

通过标准：解析、预览、草稿和重复合并分开执行，任何取消都不提前写库存。

收尾：删除 F5-A 临时库存记录。

## F6：BOM 列映射与版本差异

入口：首页 `对比 BOM 版本`

建立两个 UTF-8 CSV：

`bom-v1.csv`

```csv
立创编号,位号,数量,名称,厂家型号,封装
<C_A>,R1 R2,2,A,PART-A,0603
<C_B>,C1 C2,2,B,PART-B,0603
<C_C>,U1,1,C-旧,PART-C,QFN-32
```

`bom-v2.csv`

```csv
立创编号,位号,数量,名称,厂家型号,封装
<C_A>,R1 R2 R3,3,A,PART-A,0603
<C_B>,C1,1,B,PART-B,0603
<C_C>,U1,1,C-新,PART-C2,QFN-48
<C_D>,J1,1,D,PART-D,HDR
```

1. 执行入口。在同一个系统文件选择器中同时选中 `bom-v1.csv` 和 `bom-v2.csv`；该入口只打开一次选择器，并且必须正好选两份文件。
2. 映射窗口顶部应为 `1. bom-v1.csv`、`2. bom-v2.csv`，第一份固定是基准 BOM。顺序相反时点击取消，重新执行“对比 BOM 版本”，再在同一个文件选择器中同时选择两份文件；当前映射窗口不能直接调整文件顺序。
3. 分别打开两个文件页签，核对工作表、表头候选、表头行、字段映射和原始行预览。CSV 的“工作表”只有一个与文件对应的选项，这是正常结果。
4. 故意把同一原始列映射到两个字段，确认界面提示并禁用提交；修正后继续。
5. 差异应为全部 `4`、新增 `1`、移除 `0`、数量增加 `1`、数量减少 `1`、身份变化 `1`；逐个切换筛选核对可见行。
6. 恢复“全部变化”，点击“导出差异 CSV”。导出始终包含完整差异，不是当前筛选后的子集。
7. 编码边界可单独测试：把 `bom-v1.csv` 另存为内容相同的“UTF-16 LE with BOM”或 GB18030 文件，与 UTF-8 原文件同时选择并完成映射，预期差异为 `0`，中文和格式栏编码正确。UTF-16 文件没有 BOM 不属于支持用例。

通过标准：差异类型、计数和前后数量正确；整个过程不修改库存。

## F7：BOM 整批出库与撤销

入口：首页 `按 BOM 整批出库` 和 `BOM 出库批次与撤销流水`

### 前置条件

每次测试都使用两个从未参加过 BOM 出库的新临时库存：`F7-A` 真实数量 `10`、`F7-S` 真实数量 `6`，各使用一个新的有效 C 编号。用“按立创 C 编号添加”创建，名称必填，模型不要求绑定。先另存库存备份。

建立 `bom-stockout.csv`：

```csv
立创编号,位号,数量,名称
<C_F7_A>,R1,1,F7-A
<C_F7_S>,R2,1,F7-S
```

1. 执行“按 BOM 整批出库”，选择文件；先完成列映射，再输入板数 `2`，最后填写出库备注并确定，备注可以留空但不能点取消。
2. 预览应显示 F7-A 从 `10` 扣 `2` 后为 `8`，F7-S 从 `6` 扣 `2` 后为 `4`。
3. 在预览点击“确认整批出库”，随后在预览面板内部的确认弹窗再次点击“确认整批出库”。库存应一次变为 `8/4`。
4. 打开或刷新“BOM 出库批次与撤销流水”，核对新增一个批次和两条出库流水。
5. 再次从头导入同一文件。出现“是否作为新生产批次出库”时取消，流程应结束，库存保持 `8/4`。
6. 第三次从头导入同一文件，在重复语义确认中选择作为新生产批次；随后仍会打开预览，必须再次提交并确认整批出库。库存应变为 `6/2`，流水面板新增第二批和两条流水。
7. 此时尝试删除 F7-A，应提示仍被未撤销批次引用，记录不被删除。
8. 在流水面板撤销第二批：确认撤销后会出现“撤销备注”输入框，留空也要点确定。库存应回到 `8/4`，并新增两条反向流水。
9. 撤销第一批，库存应回到 `10/6`。两个批次都显示已撤销，且“撤销”按钮消失。UI 无法再次点击已撤销批次，重复撤销防护由单元测试覆盖。

通过标准：出库和撤销都是整批成功或整批失败；撤销保留原流水并新增反向流水。

收尾：确认两批均已撤销，再删除 F7-A、F7-S 两条临时库存。历史批次不会自动删除；若不需要保留本次历史，可恢复出库前备份，再删除两条临时库存。下次手测仍应换新的临时记录。

## F8：项目需求与采购

入口：首页、SCH 或 PCB 的 `项目需求与采购`。面板可从三个位置打开，但“捕获当前设计”必须在已打开的 SCH 或 PCB 中执行。

### 前置条件

1. A=`真实 10`、B=`真实 1`，无同 C 编号重复库存。执行“另存库存备份”保存 `f8-before.json`；该备份会用于恢复原有项目快照和采购记录。
2. 打开“项目需求与采购”的“需求快照”，逐条删除现有快照，直到列表为空。精确数值测试必须隔离全部旧快照，因为采购建议会聚合所有快照；不愿临时清空时，本节不能按下方绝对数值判定。
3. 新建一份从未捕获过的可编辑测试 SCH，至少有两个图页；从 EDA 自带器件库放置 `<C_A>` 模型两个、`<C_B>` 模型一个，不要求通过插件库存放置。
4. 打开“采购记录”，记录测试前 CNY 的总数量、总金额和记录数；没有 CNY 记录时三项基线均按 `0` 处理。

### 测试步骤

1. 从该 SCH 打开项目面板，在“需求快照”点击“捕获当前设计”，名称填 `测试板`，板数填 `3`。
2. 需求快照表只显示快照名称、UUID、文档类型、板数、捕获时间和物料行数，不显示逐物料需求。切换到“采购建议”，应只看到缺料的 B：总需求 `3`、现有 `1`、建议采购 `2`；A 库存充足，因此不显示。
3. 回到“需求快照”，对 `测试板` 执行“修改板数”，填 `4` 并确认。再看“采购建议”，B 应为总需求 `4`、建议采购 `3`；A 仍不显示。
4. 回到画布再增加一个 A。重新点击“捕获当前设计”，填写同一名称和板数 `4`；第一次在差异确认中取消，旧快照和采购建议不变。
5. 再次捕获并确认差异。采购建议中 A 现在应出现：总需求 `12`、现有 `10`、建议采购 `2`；B 仍为总需求 `4`、建议采购 `3`。
6. 切换到同一 SCH 的另一图页再次捕获。宿主返回同一文档 UUID 且读取到相同 BOM 时，应提示“当前设计、板数和快照名称均未变化”，快照数量仍为 `1`。若出现不同 BOM 的差异确认，点击取消并记录差异；若返回新 UUID 并新增快照，记录两个 UUID 后删除新增快照；若提示空设计，记录提示。继续下一步前，必须确认只剩一条 `测试板`，板数为 `4`，采购建议仍为 A 总需求 `12`、B 总需求 `4`。
7. 切换到“采购记录”，点击“登记采购”，填写 C 编号 `<C_A>`、名称 `测试-A`、数量 `3`、单价 `125`、币种 `CNY`、日期 `2026-07-22`，并在“关联需求快照”选择 `测试板`。
8. 保存后，该采购行的合计列应显示 `375 CNY`；上方“按币种汇总”中的 CNY 总数量应比基线增加 `3`、总金额增加 `375`、记录数增加 `1`。这里的 `125` 和 `375` 都是最小货币单位，不代表 125 元和 375 元。
9. 关闭并重开面板，采购日期仍为 `2026-07-22`。删除该采购记录，成本汇总同步更新；登记采购不会增加库存。
10. 在“采购建议”点击“导出采购建议”，核对 A、B 的需求、库存、缺口和建议数量。

通过标准：快照重同步必须先确认，采购数据持久化且不修改库存。同一物理板同时保留 SCH、PCB 两份快照会重复累计，是当前已知限制。

收尾：恢复 `f8-before.json`，确认测试前的项目快照、采购记录和 A/B 库存均已恢复。

## F9：库存元数据与替代料

### F9-A：元数据与显示列

入口：`库存总览 -> A 所在行 -> 操作 -> 完整编辑`

1. 记录 A 原有元数据，并记录“设置显示列”中九个可配置列测试前的勾选状态。进入“完整编辑”，设置收藏、数据手册 `https://example.com/test.pdf`、自由位置和柜/盒/行/列后保存。
2. 回到 A 的“操作”列点击“查看详情”，核对字段都已保存；详情中的数据手册只是 URL 文本。关闭详情后，在总览操作列点击 `↗` 图标（悬停提示“打开数据手册”），确认由宿主打开链接。
3. 分别尝试保存 `javascript:alert(1)`、`ftp://example.com/a.pdf`、`https://user:pass@example.com/a.pdf`，都应被拒绝且原数据不变。
4. 点击总览顶部带 `☷` 图标、悬停提示为“设置显示列”的按钮。可配置项应为：C 编号 / 厂家型号、库存、封装、EDA 模型、仓位、分类、补货状态、最低库存、更新时间；名称和操作固定不可隐藏。
5. 隐藏任意几个可配置列，关闭重开总览，设置应保留。
6. 在“库存、最低库存、补货状态”中先取消任意两项；剩下最后一项应变为禁用并保持显示，不会弹错误。
7. 点击“恢复默认列”，业务列应依次为：名称、操作、编号、库存、封装、EDA 模型、仓位、分类、补货状态、最低库存、更新时间。

通过标准：元数据独立持久化；显示列配置持久化；库存信息列至少保留一项。

收尾：恢复 A 测试前的元数据，并按记录逐项恢复九个可配置列的勾选状态。

### F9-B：替代料

入口：首页、SCH 或 PCB 的 `管理替代料关系`

前置条件：B=`真实 1`。若 S 尚不存在，选择一个新的有效 C 编号，通过“按立创 C 编号添加”建立名称 `测试-S`、真实数量 `6` 的记录，模型不要求绑定。B、S 的 C 编号不同，无现有替代关系，也没有未撤销 BOM 出库批次。

1. 执行入口，依次选择 B、需求数量 `5`、“新增已确认替代料关系”、S；在备注框填 `F9手测` 并确定，再在最终确认框确定建立关系。
2. 重新执行入口，选择 B、需求 `5`。候选列表中 S 应显示“精确库存且足量”；此时取消选择，不进入删除确认。
3. 核对 B、S 仍为 `1/6`，建立关系不会改库存。
4. 从 EDA 自带器件库把 `<C_B>` 模型放入一份只含一个 B 的设计，不要求从插件库存放置。执行 `元器件库存 -> 校验当前设计库存`，板数填 `2`，应按 B 自身库存报告不足 `1`，不能自动使用 S。
5. 建立一份只含 B、单板数量 `1` 的 BOM，执行“按 BOM 整批出库”，完成列映射、板数 `2` 和备注确认。流程应在出库预检阶段提示库存不足并结束，不应打开可提交的预览，也不能扣 S。
6. 在总览尝试删除 B，应提示仍被替代料关系引用。
7. 验证合并保护：对 B 执行“完整编辑”，临时把 C 编号改成 `<C_S>` 并保存；重复比较出现后点击“确认合并”，应提示替代料引用阻止合并，B、S 两条记录及原 C 编号均不变。
8. 删除关系：重新执行“管理替代料关系”，选择 B、需求 `5`，在候选列表选择 S，再在删除确认框确定。B、S 的数量和其他字段应保持不变。

通过标准：替代关系只供人工确认，不参与 F1 自动匹配或 F7 自动扣减；被引用记录不能删除或作为合并中会消失的记录。

收尾：确认 B、S 的替代关系已删除，数量为 `1/6`；S 是本节专用临时记录时再将其删除。
