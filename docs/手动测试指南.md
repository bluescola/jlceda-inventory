# 0.5.1 新功能测试方法

本文只测试 `0.5.0` 新增并由 `0.5.1` 完成宿主兼容修复的 P0、F1-F9，不包含安装、旧功能回归、发布要求和跨设备测试。

每个功能可以单独测试。涉及恢复或扣库存的 P0、F7 开始前，先使用“导出库存备份”保存当前库存。

开始某一节前，只需把该节使用的 A、B、C、D、S 数量恢复到下表数值。

## 公共测试数据

准备以下库存，C 编号使用你测试工程中互不相同的真实编号：

| 代号 | 数量 |
| --- | --- |
| A | 准确 `10` |
| B | 准确 `1` |
| C | 大概 `2` |
| D | 未知 |
| S | 准确 `6`，作为替代料 |
| M | 只存在于设计/BOM，不加入库存 |

下面用 `<C_A>`、`<C_B>` 等表示这些编号，建立测试文件时替换为实际 C 编号。

## P0：自动备份与恢复

入口：`元器件库存 -> 自动备份设置 / 恢复库存备份`

1. 导出当前库存备份，记住 A 的数量为 `10`。
2. 把 A 改成 `7`，再恢复刚才的备份，A 应恢复为 `10`。
3. 再次进入恢复，选择“上次恢复前快照”，A 应回到 `7`。
4. 桌面端第一次打开“自动备份设置”，不应出现启用选择、路径输入或覆盖确认。插件应直接在系统“文档”目录下创建 `JLCEDA-Inventory/jlceda-inventory-latest.json`，文件内容应为 schema-v9，并显示最终完整路径；若提示无法读取目录，只需在扩展管理器开启“允许外部交互”后重试。
5. 如果原来保存过其他自动备份路径，再次打开设置并选择“改用系统默认位置”，应一键迁移到上述默认位置，不要求输入路径。
6. 修改 B，自动备份文件应随之更新，且不弹窗。
7. 再次打开设置，依次检查“立即测试备份”和“停用自动备份”。停用后修改库存不应更新文件，重新启用应继续使用当前文件。
8. Web 端打开自动备份设置，应提示不支持固定路径；手动导出和恢复仍可使用。

通过标准：用户无需填写路径或手建 JSON；首次设置没有多余菜单和确认；恢复结果完整，取消没有副作用，自动备份失败不能影响主库存保存。

## F1：当前设计与外部 BOM 库存校验

入口：SCH/PCB 的 `校验当前设计库存`，或首页的 `校验外部 BOM 库存`

1. 在测试设计中放置：A 两个，B、C、D、M 各一个，再放置一个没有 C 编号的器件。
2. 另放一个器件并设置为“不加入 BOM”。
3. 执行库存校验，板数输入 `2`。
4. 检查结果：
   - A：需求 `4`，库存充足。
   - B：需求 `2`，库存不足 `1`。
   - C：需复核。
   - D：需盘点。
   - M：库存缺失。
   - 无 C 编号器件：身份未匹配。
   - 不加入 BOM 的器件：不出现在结果中。
5. 切换状态筛选并导出缺料 CSV。
6. 把相同数据保存成 CSV，通过“校验外部 BOM 库存”完成表头和列映射，再核对一次结果。

通过标准：SCH、PCB、外部 BOM 的结果一致；校验前后所有库存数量不变。

## F2：最低库存与补货筛选

入口：`库存总览 -> 完整编辑` 和总览顶部“补货状态”

1. 给 A、B、C、D 分别设置最低库存 `3`。
2. 检查状态：A 充足，B 低库存，C 可能需要补货，D 需要盘点。
3. 把一条库存改为 `0`，应显示已用完。
4. 选择“需要补货”，应看到 B、C 和已用完记录。
5. 选择“需要盘点”，应只看到 D。
6. 导出补货清单，核对准确、大概、未知数量没有被混为同一种状态。

通过标准：最低库存只影响提示和筛选，不会自动改变当前数量。

## F3：定位选中元器件库存

入口：SCH/PCB 的 `定位选中元器件库存`

1. 在画布中只选中 A，执行命令。
2. 库存总览应自动打开，滚动到 A 并短暂高亮。
3. 同时选中 A、B，应先让你选择要定位哪个器件。
4. 选中 M，应提示没有库存，并询问是否打开预填新增表单。
5. 在新增表单取消，库存不应增加。
6. 不选任何器件时执行命令，应提示没有可识别的选中元件。

通过标准：定位目标准确；除非最终确认新增，否则库存不变。

## F4：EDA 模型候选与人工确认

入口：`库存总览 -> 无模型记录 -> 重新匹配 EDA 模型`

1. 准备一条有合法 C 编号但没有 EDA 模型的库存。如果该编号能精确命中模型，就换一个未命中的编号。
2. 点击“重新匹配 EDA 模型”。精确查询未命中后，输入一个能搜到结果的 C 编号、厂家型号或名称。
3. 核对候选中的名称、C 编号、厂家型号、封装、符号和封装模型。
4. 第一次取消选择，模型状态不应变化。
5. 再次搜索并确认一个候选，记录应变为“EDA 模型可用”。
6. 检查原库存名称、数量、分类、位置和备注没有被候选静默覆盖。

通过标准：候选只在人工确认后绑定；取消、无结果和查询失败都不修改库存记录。

## F5：包装码入库

入口：首页 `扫描包装码入库`

使用以下内容，将 `<C_A>` 替换为 A 的实际编号：

```text
{pc:<C_A>,pm:MT-PACKAGE-A,qty:5}
```

1. 粘贴包装码，核对预览中的原文、C 编号、厂家型号和数量。
2. 第一次取消预览，库存不变。
3. 再次确认预览，系统只应打开预填的新增草稿，还不能立即写库存。
4. 在草稿中保存；因为 A 已存在，应显示重复比较。
5. 取消合并时 A 仍为 `10`；确认合并后 A 应变为 `15`。
6. 分别输入 `pm:ABC,qty:1`、`pc:C123,qty:0`、`pc:C123,pc:C456,qty:1`，均应在打开草稿前报错。

通过标准：解析、草稿、重复合并分为三个明确阶段，任何取消都不提前写库存。

## F6：BOM 列映射与版本差异

入口：首页 `对比 BOM 版本`

建立两个 UTF-8 CSV，并替换 C 编号：

`bom-v1.csv`

```csv
立创编号,位号,数量,名称,厂家型号,封装
<C_A>,R1 R2,2,A,PART-A,0603
<C_B>,C1 C2,2,B,PART-B,0603
<C_C>,U1,1,C-旧,PART-C,QFN-32
```

`bom-v2.csv`

```csv
立创编号,位号,数量,名称,厂家型号,封装
<C_A>,R1 R2 R3,3,A,PART-A,0603
<C_B>,C1,1,B,PART-B,0603
<C_C>,U1,1,C-新,PART-C2,QFN-48
<C_D>,J1,1,D,PART-D,HDR
```

1. 按顺序选择 v1、v2。
2. 在两份文件中核对工作表、表头、原始预览和列映射。
3. 故意把同一列映射给两个字段，应阻止提交；修正后继续。
4. 差异报告应显示：A 数量增加、B 数量减少、C 身份变化、D 新增。
5. 分别切换变化类型筛选并导出差异 CSV。
6. 可再把同一 CSV 保存为 UTF-16 或 GB18030，确认中文和映射结果一致。

通过标准：差异类型和前后数量正确；整个过程不修改库存。

## F7：BOM 整批出库与撤销

入口：首页 `按 BOM 整批出库` 和 `BOM 出库批次与撤销流水`

建立 `bom-stockout.csv`：

```csv
立创编号,位号,数量,名称
<C_A>,R1,1,A
<C_S>,R2,1,S
```

1. 确认开始前 A=`10`、S=`6`。
2. 选择文件，板数输入 `2`，完成映射和预览。
3. 预览应显示 A 扣 `2` 后为 `8`，S 扣 `2` 后为 `4`。
4. 确认整批出库，库存应一次变为 A=`8`、S=`4`，并生成一个批次和两条流水。
5. 再次导入相同需求，取消“新的生产批次”，数量不变。
6. 再次执行并确认是新生产，数量变为 A=`6`、S=`2`，新增第二个批次。
7. 存在未撤销批次时尝试删除 A，应被阻止。
8. 在流水窗口撤销第二批，数量回到 `8/4`；再撤销第一批，数量回到 `10/6`。
9. 重复撤销同一批次，应被阻止。

通过标准：出库和撤销都是整批成功或整批失败；撤销保留原流水并新增反向流水。

## F8：项目需求与采购

入口：首页、SCH 或 PCB 的 `项目需求与采购`

1. 在测试 SCH 中放置 A 两个、B 一个。
2. 打开项目面板，捕获当前设计，名称填 `测试板`，板数填 `3`。
3. 快照应显示 A 需求 `6`、B 需求 `3`；A 库存足够，B 建议采购 `2`。
4. 修改板数为 `4`，需求应更新为 A=`8`、B=`4`，B 建议采购 `3`。
5. 在设计中再增加一个 A，重新捕获；第一次取消差异确认，旧快照不变；再次确认后 A 的单板用量变为 `3`。
6. 在同一原理图的另一图页再次捕获，应更新同一快照，不能新增一份相同原理图快照。
7. 登记采购：数量 `3`、单价 `125`、币种 `CNY`、日期 `2026-07-22`，成本汇总应为 `375 CNY`。
8. 关闭重开，采购日期仍应是 `2026-07-22`；删除采购记录后成本同步更新。
9. 导出采购建议 CSV，核对需求、库存、缺口和建议数量。

通过标准：快照重同步必须先确认，采购记录不自动增加库存。同一物理板同时保留 SCH、PCB 快照会重复累计，这是当前已知限制。

## F9：库存元数据与替代料

入口：库存新增/完整编辑、总览“设置显示列”，以及首页 `管理替代料关系`

### 元数据

1. 编辑 A，设置收藏、数据手册 URL、自由位置和柜/盒/行/列。
2. 保存后核对这些字段同时存在，数据手册能通过宿主打开。
3. 输入 `javascript:`、非 HTTP(S) 或含账号密码的 URL，应被拒绝。
4. 在总览隐藏几个可配置列，关闭重开后设置应保留。
5. 尝试隐藏全部数量相关列，系统必须至少保留一个库存信息列。
6. 点击“恢复默认列”完成清理。

### 替代料

1. 选择 B 为基准，需求量填 `5`，选择 S 为替代候选并确认。
2. 再次打开 B，S 应显示为已确认候选，状态为“精确库存且足量”。
3. B、S 数量不能因为建立关系而变化。
4. 用 B 执行 F1/F7，即使 S 足量，仍应按 B 自身库存报告不足或阻止出库，不能自动扣 S。
5. 尝试删除 B 或把 B 作为会消失的合并源，应被阻止。
6. 删除替代关系，B、S 的数量和其他字段保持不变。

通过标准：元数据可独立编辑并正确持久化；替代关系只供人工参考，不参与自动匹配和扣减。
